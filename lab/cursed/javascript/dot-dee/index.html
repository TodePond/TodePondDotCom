Everything is an object

In javascript, everything is an Object.

So anything you add to Object.prototype gets added to everything. 

    Reflect.defineProperty(Object.prototype, "d", {
      get() {
        const value = this.valueOf();
        console.log(value);
        return value;
      },
    });
  
Object.prototype.name = "Luke"

console.log("Test".name)
console.log((1234).name)
console.log([3, 2, 5].name)
console.log({age: 31}.name)

delete Object.prototype.name

    Reflect.deleteProperty(Object.prototype, "d")


It's not a good idea to add things directly to Object.prototype because it messes up looping through objects.

It'll loop through the object's keys, but also through the keys you added to the prototype, which is probably not what you want because it'll mess up a lot of stuff, including other libraries. 

    Reflect.defineProperty(Object.prototype, "d", {
      get() {
        const value = this.valueOf();
        console.log(value);
        return value;
      }
    })
  
Object.prototype.name = "Luke"

for (const key in { age: 31 }) {
  console.log(key)
}

delete Object.prototype.name

    Reflect.deleteProperty(Object.prototype, "d")


It's better to use Reflect.defineProperty instead. It still adds the property to everything. 

Reflect.defineProperty(Object.prototype, "name", {
  value: "Luke"
})

console.log("Test".name)
console.log((1234).name)
console.log([3, 2, 5].name)
console.log({age: 31}.name)

    Reflect.deleteProperty(Object.prototype, "name")


And it doesn't mess up looping.

Reflect.defineProperty(Object.prototype, "name", {
  value: "Luke"
})

for (const key in { age: 31 }) {
  console.log(key)
}

    Reflect.deleteProperty(Object.prototype, "name")


This is because Reflect.defineProperty makes properties non-enumerable by default. You can override this though. 

Reflect.defineProperty(Object.prototype, "name", {
  value: "Luke",
  enumerable: true
})

for (const key in { age: 31 }) {
  console.log(key)
}
    Reflect.deleteProperty(Object.prototype, "name")


Getters

You can also define a property as a getter.

Reflect.defineProperty(Object.prototype, "name", {
  get() {
    return "Luke"
  }
})

console.log("Test".name)
    Reflect.deleteProperty(Object.prototype, "name")


You can add side effects to the getter.

Reflect.defineProperty(Object.prototype, "name", {
  get() {
    console.log("Getting name...")
    return "Luke"
  }
})

console.log("Test".name)
Reflect.deleteProperty(Object.prototype, "name")


To make .d happen, the side effect just needs to log to the console. 

Reflect.defineProperty(Object.prototype, "d", {
  get() {
    console.log(this)
  },
})

"Hello world".d
Reflect.deleteProperty(Object.prototype, "d")


But wait! It's not working right. It's not printing Hello world. It's printing "Hello world" instead.

That's because we're printing the object itself, not the primitive value of the object.

We can get what we need by using the valueOf method.

Reflect.defineProperty(Object.prototype, "d", {
  get() {
    console.log(this.valueOf())
  },
})

"Hello world".d
Reflect.deleteProperty(Object.prototype, "d")


It's logging the right thing now, but we can't plop it in anywhere yet.

Reflect.defineProperty(Object.prototype, "d", {
  get() {
    console.log(this.valueOf())
  },
})

"Hello world".d.toUpperCase().d
Reflect.deleteProperty(Object.prototype, "d")


To fix it, we need to return the value itself in the getter

Reflect.defineProperty(Object.prototype, "d", {
  get() {
    const value = this.valueOf()
    console.log(value)
    return value
  },
})

"Hello world".d.toUpperCase().d
Reflect.deleteProperty(Object.prototype, "d")
